The development version of Library Weasel is now up on github at github.com/almibe/library-weasel. This early version is still being worked on and can be considered to be a proof of concept for the project. So far this version implements the key features of the UI with a simple two pane display and a context sensitive toolbar on top for performing actions. One pane is for displaying resources and the other is for dashes. Below are two screen shots of this UI.

Dashes are small single purpose application panels that are supplied to Library Weasel via plugins to work with resources. These are shown in a tab pane similar to modern IDEs and web browsers. Resources can represent anything a plugin creator wants to keep track of. Example resources could include a file on disk, a URL, or a reference into a database.

Currently supported resources in the github project are Groovy script files and OrientDB SQL files. These resources are made to be used with two included plugins. One is for running scripts from within the context of Library Weasel for scripting the application, and the other is for running queries and commands against an embedded OrientDB instance that comes included with Library Weasel. These two plugins create a foundation on which other plugins can be built.

As I document this application and create new plugins I'll be updating this website. For now the application can be found on github with instructions for building although I'll be posting a built release soon for download.
